Dio koji rade svi kandidati

Na dashboardu konstruisati layout i postaviti 7 angular komponenti na mjesta prema slici. Komponente snimati u u app/components folderu sa prefixom linkos- npr u folder linkos-about-user smjestiti komponentu "About user".

Opis komponenti

About company
Prikazati naziv adresu broj tel. kompanije
About user
Prikazati sliku korisnika ime prezime i ulogu u kompaniji te tri linka sa ikonama: za podesenja, poruke i profil. Primjer:
Active users - chat
Lista korisnika sa imenom i prezimenom te ulogom u kompaniji i dva linka sa ikonama za slanje poruke i slanje emaila korisniku
Company news
Kartice sa naslovom i opisom vjesti i obavjestenja imaju dvije kategorije redovna i vazna obavjestenja bojom naglasiti vazna obavjestanja. Trebaju biti sortirana po datumu da su najnovija na vrhu
Clock
Digitalni sat koji pokazuje realno vrijeme (bez refresha) sa datumom ispod
Calendar
Kalendar sa oznacenim eventima (event ima naziv vrijeme i lokaciju). Iskoristiti neki od ova dva kalendara https://github.com/mattlewis92/angular-calendar https://github.com/angular-ui/ui-calendar
Tasks
Lista zadataka za korisnika sa krticama od kojih svaka sadrzi opis zadatka, vrijeme zadatka, link detalje o tasku.

Individualni zadaci (Backend i povezivanje sa frontendom)

Vrijedi za sve: Potrebno je napraviti REST Api sa feathersjs nad mysql bazom (nad jednom tabelom). REST api povezati sa komponentom da prikazuje podatke koji se dobiju sa Api-ja

Kenan Tendzo

REST API za active users. Uvezati komponentu Active users sa ovim api-jem odnosno prikazati podatke koji se dobiju sa api-ja.

Denis Dzafo

REST API za Company news. Uvezati komponentu Company news sa ovim api-jem odnosno prikazati podatke koji se dobiju sa api-ja.

Kerim Dragolj

REST API za Calendar. Uvezati komponentu Calendar sa ovim api-jem odnosno prikazati podatke koji se dobiju sa api-ja.

Faris Cancar

REST API za Tasks. Uvezati komponentu Tasks sa ovim api-jem odnosno prikazati podatke koji se dobiju sa api-ja.

Sabahudin Brkan

REST API za About user i About company koji vraca podatke za komponente About user i about company Uvezati komponente sa podacima koji se dobiju sa API-ja.
